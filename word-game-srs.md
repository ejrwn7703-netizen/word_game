# 단어 맞추기 게임 - SRS (Software Requirements Specification)

작성일: 2026-06-17  
버전: 0.1 (초안)

---

## 1. 프로젝트 개요

실시간 멀티플레이어 단어 맞추기 웹 게임.  
방을 만들고 링크를 공유해 친구들과 함께 접속, 화면에 뜨는 힌트를 보고 단어를 먼저 맞추는 게임.  
추후 다양한 게임 모드를 추가할 수 있도록 확장 가능한 구조로 개발.

---

## 2. 사용자 유형

| 유형 | 설명 |
|------|------|
| 방장 | 방을 생성하고 게임 시작 권한을 가진 플레이어 |
| 참여자 | 링크 또는 방 코드로 입장한 플레이어 |

---

## 3. 기능 요구사항

### 3.1 닉네임 입력
- 사이트 접속 시 닉네임 입력 화면 표시
- 로그인 / 회원가입 없음
- 닉네임 입력 후 → 방 만들기 또는 방 코드 입력으로 진행

### 3.2 방 시스템
- **방 만들기**: 고유한 방 코드(예: ABC123) 및 입장 링크 자동 생성
- **방 입장**: 링크 클릭 또는 방 코드 직접 입력
- **인원 제한**: 없음
- **대기실**: 입장한 참여자 닉네임 목록 실시간 표시
- **게임 시작**: 방장만 시작 버튼 클릭 가능

### 3.3 힌트 시스템

| 시점 | 내용 |
|------|------|
| 0초 (첫 힌트) | 앞 2글자 또는 뒤 2글자 공개 (랜덤) |
| 15초 (두 번째 힌트) | 단어 카테고리 공개 (예: `힌트: 역사 인물`) |
| 20초 | 타임아웃 → 정답 공개 후 다음 문제로 |

**힌트 예시 (세종대왕 - 4글자)**
- 앞 공개: `세종 ㅇ ㅇ`
- 뒤 공개: `ㅇ ㅇ 대왕`

**글자 수가 다른 경우 예시 (대한민국 - 4글자 / 김치찌개 - 4글자)**
- 앞 공개: `대한 ㅇ ㅇ` / `김치 ㅇ ㅇ`
- 뒤 공개: `ㅇ ㅇ 민국` / `ㅇ ㅇ 찌개`

### 3.4 게임 진행
- 단어 목록에서 문제 랜덤 출제 (카테고리 구분 없이 섞어서)
- 가장 먼저 정답을 입력한 플레이어가 **+1점** 획득
- 오답은 점수 변화 없음 (패널티 없음)
- 정답 맞춘 즉시 누가 맞췄는지 전체 공개 후 다음 문제로 이동
- **승리 조건 (둘 중 하나를 먼저 충족하면 게임 종료)**
  - 먼저 10점 획득한 플레이어 승리
  - 게임 시작 후 **5분 경과** 또는 **30문제 소진** 시 → 가장 높은 점수의 플레이어 승리

### 3.5 점수판
- 게임 화면에 실시간 점수판 항상 표시
- 점수 높은 순으로 정렬

### 3.6 게임 종료
- 10점 달성 시 즉시 게임 종료
- 승자 닉네임 및 최종 점수판 표시
- **다시 하기** 버튼: 방장만 클릭 가능 → 대기실로 복귀

## 4. 화면 구성

```
[1] 메인 화면
    └─ 닉네임 입력
    └─ 방 만들기 버튼
    └─ 방 코드 입력 + 입장 버튼

[2] 대기실
    └─ 방 코드 / 초대 링크 표시 (복사 버튼)
    └─ 참여자 목록 (실시간)
    └─ 게임 시작 버튼 (방장만)

[3] 게임 화면
    └─ 힌트 표시 영역 (크게)
    └─ 타이머 (20초 카운트다운)
    └─ 정답 입력창 + 제출 버튼
    └─ 실시간 점수판
    └─ 정답/오답 로그 (채팅 형식)

[4] 결과 화면
    └─ 승자 표시
    └─ 최종 점수판
    └─ 다시 하기 버튼 (방장만)
```

## 5. 기술 스택 (예정)

| 구분 | 기술 | 이유 |
|------|------|------|
| 프론트엔드 | HTML / CSS / JavaScript | 별도 설치 없이 브라우저에서 실행 |
| 백엔드 | Node.js + Socket.io | 실시간 통신 (여러 명이 동시에 플레이) |
| 단어 데이터 | JSON 파일 | 단어 추가/수정이 쉬움 |
| 호스팅 | Render 또는 Railway | 무료로 인터넷 배포 가능 |
